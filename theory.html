3. What is an array?
An array is a collection of items of the same type stored at contiguous memeory locations. Each element within the array can be accessed via its positions by adding an offset to a base value. This base value usually is the frist element of the array and has index 0.
4. What are arrays useful for?
Arrays are useful when one needs to use many variables of the same type and store a collection of data. Arrays help maintain large sets of data by using a single variable name to avoid confusion and organize the various data elements in a clear and efficient way. For these reasons, arrays are used for matrix operations and to implement structures like queues, heaps, stacks and hash tables.

6. How do you change an array element?
To change an array element and replace it with another one you must:
  1. use a for loop to iterate as much as array.length
  2. in each iteration check where is the array element to be replaced
  3. replace the element at that index and break out of the loop
If you want to change the banana element with strawberry in this array:

const fruits = ['Apple', 'Banana'];


for( let index = 0; index < fruits.length; index++){
if(fruits[index] === 'Banana'){
fruits[index] = 'Stawberry';
break;
}
}
console.log(fruits); // ['Apple', 'Stawberry']

    The DOM Events has 3 phases of event propagation:
    • Capturing phase
    • Target phase
    • Bubbling phase
    Bubbling is a process where the events "bubble" from the inner element up through parents like a bubble in a water.
    Capturing is a process where the event goes down to the element. The capturing option has two possible values:
    1. False : the handler is set on the bubbbling phase
    2. True: the handler is set on the capturng phase

    Inside a table, clicking on <td> the event first goes through the ancestors chain down to the element (capturin phase), then it reaches the target and triggers there (target phase), and then it goes up again (bubbling phase), calling handlers on its way.
